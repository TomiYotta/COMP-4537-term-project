# Backend AI Service

This service exposes an API to evaluate a student's answer against a reference answer using sentence embeddings from Hugging Face (`Xenova/all-MiniLM-L6-v2`).

## Will it work on other machines if files are in `.gitignore`?

Yes.

Ignored files like `node_modules`, `.env`, and `.cache` are machine-local/generated files.
Other machines recreate them after pulling the repository.

- `node_modules/`: regenerated by `npm install`
- `.cache/`: regenerated automatically on first model load
- `.env`: each machine creates its own local config

## Requirements

- Node.js 18+
- npm

## Setup on a new machine

1. Open a terminal in the `backend` folder.
2. Install dependencies:

```bash
npm install
```

3. Create `.env` file in `backend/`:

```env
PORT=3000
CLIENT_ORIGIN=http://localhost:5173
```

4. Start the server:

```bash
cd backend
npm run dev
```

## First run behavior

On first startup, the model files download and are cached in `backend/.cache`.
This can take longer than later runs.

## API Endpoints

Base URL: `http://localhost:3000`

### Health Check

- Method: `GET`
- URL: `/api/ai/health`

Example:

```bash
curl http://localhost:3000/api/ai/health
```

### Evaluate Answer

- Method: `POST`
- URL: `/api/ai/evaluate`
- Content-Type: `application/json`

Request body:

```json
{
  "question": "What is photosynthesis?",
  "referenceAnswer": "Photosynthesis is the process by which plants convert light energy into chemical energy.",
  "studentAnswer": "Plants use sunlight to make food through photosynthesis."
}
```

PowerShell example:

```powershell
$body = @{
  question = 'What is photosynthesis?'
  referenceAnswer = 'Photosynthesis is the process by which plants convert light energy into chemical energy.'
  studentAnswer = 'Plants use sunlight to make food through photosynthesis.'
} | ConvertTo-Json

Invoke-RestMethod -Method Post -Uri http://localhost:3000/api/ai/evaluate -ContentType 'application/json' -Body $body
```

## Common issue

If you open `/api/ai/evaluate` in a browser, it fails because browsers send `GET` by default.
Use `POST` with JSON body in Postman or code.
